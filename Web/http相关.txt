
TCP是底层通讯协议，定义的是数据传输和连接方式的规范
HTTP是应用层协议，定义的是传输数据的内容的规范
HTTP协议中的数据是利用TCP协议传输的，所以支持HTTP也就一定支持TCP      

HTTP 1.0
	每个资源都要求建立一次单独的TCP连接

HTTP 1.1
	连接的复用，可以在一次TCP连接中处理多个请求，不需要每次新建一个TCP连接请求一个资源
	问题：
		一次tcp连接中多个请求的顺序不能乱(文本传输协议,文本内容当然不能乱),当其中一个请求失败，整个请求阻塞(头部阻塞问题)

HTTP 2.0
	将"文本传输协议"改为"二进制协议",一次tcp连接中多个请求的顺序可以相互穿插，不再固定
	gRPC使用的就是HTTP 2.0

HTTP 2.0 + push
	请求建立后，服务器将你需要的资源全部push给你

HTTP 3.0
	不再使用TCP协议建立连接，使用UDP(不可靠)，但为了保证数据传输的可靠性，
	将多个请求的数据，算出一个FEC(前项错误纠正)校验值来，即使有一个请求的数据失踪，也可以通过校验值'恢复'

	-- 以前那种VCD,DVD碟片机支持纠错时，本来花了的碟片也能正常观看
